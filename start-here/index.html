Recommended Learning Flow

Start with governance, then move into security engineering. This creates the cleanest learning chain: understand authority, risk, and evidence first; then learn how to engineer the AI system safely.

1. AI Governance Command Center Start here to learn agent authority, prompt injection risk, retrieval boundaries, human approval, audit evidence, and cost governance. 2. AI Security Engineering L2 Track Continue here to learn secure AI architecture, prompt boundaries, tool permissions, retrieval security, output safety, runtime guardrails, abuse controls, and testing evidence.
3. Next Recommended Track After both AI tracks, continue into the next role-aligned path: cloud security operations, compliance evidence, or platform security engineering.

Role-Based Learning Chains

Choose the path that matches your current role or the role you are building toward.

Executive / Risk Leader Start with AI Governance Command Center, focus on authority, approval, evidence, cost control, and audit-ready summaries.
Cloud Security Engineer Start with AI Governance, then complete AI Security Engineering to understand secure architecture, tool permissions, retrieval, output policy, runtime guardrails, and testing.
GRC / Auditor / Compliance Start with audit evidence, traceability, human approval, governance boundaries, and production quality gate evidence.
Builder / Architect Start with Secure AI Application Architecture, then work through prompt, tool, retrieval, output, runtime, abuse, and testing modules.
