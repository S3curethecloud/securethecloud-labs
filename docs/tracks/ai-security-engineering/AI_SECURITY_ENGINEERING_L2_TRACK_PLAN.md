# Phase 86 — AI Security Engineering L2 Track Plan

Status: Planning Baseline

## Purpose

Plan the AI Security Engineering L2 Track as the next heavy AI intermediate learning path after the AI Governance Command Center track.

The AI Governance Command Center track teaches how to govern AI risk.

The AI Security Engineering track should teach how to build and secure AI systems.

Relationship to AI Governance Command Center

Should this AI workflow be allowed, approved, blocked, escalated, or audited?

The AI Security Engineering track answers:

How do we design the AI workflow so those controls are enforceable and testable?
