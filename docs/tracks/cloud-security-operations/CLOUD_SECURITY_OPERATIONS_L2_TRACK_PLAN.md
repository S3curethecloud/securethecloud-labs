# Cloud Security Operations L2 Track Plan

Status: Planned / Not Implemented

## Purpose

The Cloud Security Operations L2 Track is the recommended downstream path after the AI Governance Command Center and AI Security Engineering L2 Track.

This track teaches how to operate cloud security workflows in a practical, evidence-driven way.

Learners should understand how to detect, triage, explain, and evidence cloud security events without overstating production enforcement or claiming live SOC operation.

## Position in Learner Flow

Recommended sequence:

```text
Start Here
-> AI Governance Command Center
-> AI Security Engineering L2 Track
-> Cloud Security Operations L2 Track
-> Portfolio / Capstone Evidence Guide
Track Theme

Cloud security operations connects security theory to daily operational work:

cloud event
-> signal classification
-> triage
-> evidence collection
-> risk explanation
-> escalation
-> response narrative
-> portfolio-ready artifact
Intended Audience
Cloud security learners
SOC analysts moving into cloud
GRC and audit learners who need operational evidence literacy
Cloud engineers learning security operations
Architects who need detection and response fluency
Portfolio builders preparing real-world security narratives
Track Outcome

By the end of the track, learners should be able to:

explain cloud security operations workflows
classify cloud events and signals
triage IAM and workload activity
identify suspicious control-plane behavior
gather evidence without changing runtime systems
write incident-ready summaries
distinguish detection from enforcement
explain what happened, why it matters, and what evidence supports the conclusion
Governance Boundary

This planning phase does not implement a live SOC, SIEM integration, cloud provider integration, ticketing integration, alert pipeline, backend API, runtime detector, customer data workflow, or production enforcement.

The future track is educational unless a later governed phase explicitly authorizes additional scope.
