- Track: Enterprise Agent Developer L2
- Subtitle: LLM & Agentic Workflows
- Domain: Enterprise Agentic AI
- Audience: backend developers, AI application engineers, cloud architects, platform engineers, security architects, technical consultants, and senior delivery leads
- Planned modules: 9
- Runtime: read-only learning
- Backend exposure: false
- Production enforcement claim: false

## Role Thesis

The enterprise Agent Developer role is an end-to-end delivery role. It combines software engineering, LLM application architecture, enterprise integration, policy and Responsible AI, observability, cloud delivery, consulting, project execution, and mentoring.

The learner should finish the track able to explain and design the complete path from client discovery to a governed, observable, production-ready implementation plan.

## Module Progression

### Module 1 — Enterprise Agent Developer Role Architecture

Understand the full role operating model and create a role-architecture package for a synthetic enterprise assistant.

### Module 2 — Task-Oriented and Conversational Agent Design

Design prompts, state, memory, function calls, schemas, agent loops, refusal paths, and recommendation-versus-execution boundaries.

### Module 3 — Enterprise RAG and Tool-Use Engineering

Design ingestion, indexing, retrieval, reranking, grounding, source authority, tool contracts, and evidence-backed response behavior.

### Module 4 — Deterministic Routing and Multi-Agent Orchestration

Design router, supervisor, specialist-agent, handoff, retry, loop-control, escalation, and fallback patterns.

### Module 5 — Enterprise Gateway, MCP, Registry, and API Contracts

Design enterprise and local gateway integration, MCP adapters, persistent registry records, OpenAPI and JSON Schema contracts, discovery, compatibility, and versioning.

### Module 6 — OPA Policy Enforcement and Responsible AI Controls

Design policy inputs and decisions for identity, data residency, rate limits, approvals, provenance, explainability, bias review, exceptions, and audit evidence.

### Module 7 — OpenTelemetry, SLO, Cost, and Performance Engineering

Design end-to-end traces, metrics, logs, quality signals, token and cost telemetry, caching, model fallback, latency budgets, error budgets, and operational runbooks.

### Module 8 — Cloud Delivery, Containers, CI/CD, and Platform Collaboration

Package and release agents on enterprise AI platforms using containers, secrets boundaries, pipelines, evaluation gates, deployment stages, rollback, and platform-team contracts.

### Module 9 — Senior Consultant Enterprise Agent Delivery Capstone

Lead a synthetic engagement from discovery through architecture, governance, delivery plan, executive update, risk decision, team mentorship, and final evidence package.

## Capstone Outcome

The capstone learner produces:

1. client discovery summary
2. current-state and target-state process map
3. agent and multi-agent architecture
4. RAG and tool contract design
5. gateway, MCP, registry, and versioning design
6. OPA and Responsible AI control matrix
7. observability, SLO, cost, and fallback plan
8. cloud deployment and CI/CD plan
9. delivery roadmap, milestones, risks, and owners
10. executive presentation narrative
11. junior-engineer mentoring plan
12. governance boundary and residual-risk statement

## Governance Rule

No module may imply live model execution, enterprise-data access, tool invocation, gateway exposure, policy enforcement, telemetry export, cloud deployment, or production readiness unless those capabilities are explicitly implemented and production-verified.
